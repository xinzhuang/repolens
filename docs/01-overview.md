# RepoLens — 项目概述

## 一句话

帮 Agent 一次对话找到你的项目在哪。

## 核心痛点

作为一人团队，开发了很多应用，建立了很多仓库，分散在不同地方，记不住在哪里。想让编程助手去找到某一个继续开发，往往交互好几次才能找到，很低效。

## 核心洞察

问题不是"搜索代码内容"，而是 **仓库级定位** — 这个项目在哪、它是干什么的、用了什么技术栈。

## 目标用户

1. **人**：开发者自己快速找到项目
2. **AI Agent**：编程助手通过 CLI 找到项目，一次交互搞定

## 功能范围

### MVP（3天）

| 功能 | 说明 |
|------|------|
| 仓库发现 | 配置多个根目录，递归扫描 `.git`，收集元数据 |
| 指纹提取 | README 摘要、语言/框架、最近活跃时间、目录大小 |
| CLI | scan / find / list / show 四个核心命令 |
| JSON 输出 | `--json` flag，Agent 直接解析 |

### V1（+1-2天）

| 功能 | 说明 |
|------|------|
| 增量更新 | mtime 检测，只重新索引变化的仓库 |
| MCP Server | 可选，暴露 find_repo / list_repos / describe_repo |

### V2（+5-10天）

| 功能 | 说明 |
|------|------|
| 文件内容搜索 | `repolens search` 在仓库内搜索代码 |
| 语义搜索 | Embedding + 向量检索 |
| Web UI | 可选的浏览器界面 |

### 明确不做（MVP）

- 代码内容搜索
- 文件级信息
- 远程同步
- Web UI

## 技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| 语言 | **Rust** | 性能 + 单二进制 + Tantivy 原生集成 |
| 存储 | **SQLite FTS5** | 零依赖，MVP 只存元数据够用 |
| CLI 框架 | **clap** | Rust 生态标准 |
| Git 操作 | **git2** | 读取仓库信息（最后提交、分支等） |
| 文件遍历 | **walkdir + ignore** | 尊重 .gitignore，高效遍历 |

## 差异化

1. **Agent 原生设计**：JSON 输出一等公民，不是后来加的
2. **Git 语义感知**：理解仓库边界、.gitignore、分支信息
3. **仓库指纹**：不只搜名称，还搜 README、框架、语言
4. **极轻量**：单二进制，零配置即可使用
